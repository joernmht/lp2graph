# Validation: reference optima & cross-language solve fidelity

This folder holds the **third validation layer** for the extraction corpus
(`../`). The first two layers were (1) source-reference verification — every
constraint/objective resolves to a real in-bounds source line (407 refs, 0
missing) — and (2) deterministic categorization coverage. This layer actually
**solves** the formulations that can be solved here and checks that a model
expressed in one "language" yields the **same optimum** when translated to
another and handed to an independent solver.

> Note: the extraction phase forbade running/building any model. That rule was
> for faithful transcription. This validation phase runs models **on explicit
> user instruction**, using only shipped instances; it does not modify any
> upstream repo (clones in `/home/joern/milp_sources/` are read-only).

## Environment

- Python 3.12; solver packages vendored into `.deps/` via `pip install --target`
  (non-invasive — no system or venv site-packages were modified).
- **Gurobi 13.0.2** — commercial, via the user's WLS (academic) license
  (`/home/joern/gurobi.lic`; needs network to check out).
- **HiGHS** (`highspy`) — open source.
- **CBC** (bundled with **PuLP** 3.3.2) — open source.
- Determinism: single thread, solver output off, no custom heuristic seeds.
  Time limit per solver: 180 s for the hard `timtab1` MIP, 60 s otherwise.

## Reproduce

```bash
export GRB_LICENSE_FILE=/home/joern/gurobi.lic
export PYTHONPATH=$(pwd)/.deps
python3 scripts/run_experiments.py        # writes results/validation_results.json
# or solve any single portable file:
python3 scripts/solver_harness.py path/to/model.mps
```

## Experiments

| ID | What it shows | Source |
|----|---------------|--------|
| **A** | Cross-solver agreement on a dialect-neutral interchange file: Gurobi, HiGHS, CBC must report the same optimum for the same `.mps`. | repo #9 `timtab1.mps` |
| **B** | Round-trip translation fidelity: Gurobi reads the model and **re-emits** it to `.mps` and `.lp` (its writer = translation into two other textual dialects); the re-emitted files must solve to the same optimum across solvers. | repo #9 |
| **C** | From-repo Python model: build repo #8's `gurobipy` model on its smallest shipped instance, solve for the reference optimum, **export** to portable `.lp`/`.mps`, and cross-solve. This is the end-to-end "modelling-language → interchange → independent solver" check on real repository code. | repo #8 `railway-scheduling` |

Results — every solver's objective/status/runtime, plus the set of
proven-optimal objectives per experiment (`proven_optima` / `true_optimum_agreed`)
— are written to [`results/validation_results.json`](./results/validation_results.json)
and a flat [`results/summary.csv`](./results/summary.csv). Re-emitted/exported
model files land in `results/exports/` (git-ignored; regenerated by the script).

## Findings (see results/summary.csv, results/validation_results.json)

- **Cross-solver agreement (A):** Gurobi and HiGHS both prove `timtab1` optimum = **764772.0** (~1e-6). CBC returns a 180s time-limited incumbent (784722) that PuLP mislabels "Optimal" — a caveat, not a disagreement.
- **Translation fidelity (B):** Gurobi re-emits `timtab1` to `.lp` and `.mps`; both re-solve to **764772.0** under Gurobi and HiGHS. The optimum is invariant under dialect translation.
- **Big-M caveat (C):** `marcotallone` (M=1e5) — at tight tolerances native build, re-read export, and HiGHS all agree on **3913.47**; at default feasibility tolerance the same model reports a spurious **3067.03**. The objective constant (−715484.71) round-trips intact, so this is the big-M tolerance effect (the `has_big_m` flag's raison d'être), not a translation error — confirmed by re-checking the default solution's feasibility under tight tolerance.
- **Tooling caveat:** PuLP's MPS reader raises `KeyError: 'OBJ'` on Gurobi's exported objective-row name, so CBC cannot read the exported marcotallone model — a real interchange-reader incompatibility.

Note: `.deps/` (vendored solver wheels) and `results/exports/*.mps|*.lp` are build/output artifacts; consider git-ignoring them if size matters.
